## 📖 Team Workflow & Standards
To ensure we hit our program deadlines without merge conflicts, all team members must strictly adhere to our Git worflow and guidelines. 

Before you write your first line of code or open a Pull Request, you **must** review our team rulebook:
👉 **[Read the Official Contributing Guidelines](../CONTRIBUTING.md)**

### The Golden Rules:
1. **Never push to `main`.** All feature branches must branch off and merge into `develop`.
2. **Standardize Commits.** Follow Conventional Commits (e.g., `feat(api): ...`, `fix(ui): ...`).
3. **Protect Your Secrets.** All local configurations must use `.NET User Secrets` attached to the `Readar.WebApp` project.
    > ⚠️ **NOTE:** Never commit passwords or API keys to GitHub. If you accidentally push a secret, notify the entire team immediately so we can perform key rotation.
